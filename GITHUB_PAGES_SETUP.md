# Настройка GitHub Pages для EvoCon Website

## Автоматический деплой через GitHub Actions

Проект настроен для автоматической публикации на GitHub Pages при каждом push в ветку `main` или `master`.

## Шаги настройки:

### 1. Включите GitHub Pages в настройках репозитория

1. Перейдите в **Settings** вашего репозитория на GitHub
2. В левом меню выберите **Pages**
3. В разделе **Source** выберите:
   - **Source**: `GitHub Actions` (не "Deploy from a branch")
4. Сохраните изменения

### 2. Base path настраивается автоматически

✅ **Base path теперь настраивается автоматически!** 

Workflow автоматически определяет имя репозитория и устанавливает правильный base path:
- Если репозиторий `username.github.io` → base path = `/`
- Если другой репозиторий → base path = `/repo-name/`

**Вам не нужно ничего настраивать вручную!** Все пути к изображениям и ресурсам будут работать корректно на GitHub Pages.

### 3. Залейте код в репозиторий

```bash
git add .
git commit -m "Add GitHub Pages deployment"
git push origin main
```

### 4. Проверьте деплой

1. Перейдите в раздел **Actions** вашего репозитория
2. Дождитесь завершения workflow `Deploy to GitHub Pages`
3. После успешного деплоя сайт будет доступен по адресу:
   - Если репозиторий `username.github.io`: `https://username.github.io`
   - Если другой репозиторий: `https://username.github.io/repo-name/`

## Структура workflow

Workflow автоматически:
1. ✅ Устанавливает Node.js 20
2. ✅ Устанавливает зависимости (`npm ci`)
3. ✅ **Автоматически определяет base path** на основе имени репозитория
4. ✅ Собирает проект (`npm run build`) с правильным base path
5. ✅ Деплоит на GitHub Pages

**Все пути к изображениям и ресурсам автоматически адаптируются для GitHub Pages!**

## Ручной запуск деплоя

Если нужно запустить деплой вручную:
1. Перейдите в **Actions**
2. Выберите workflow `Deploy to GitHub Pages`
3. Нажмите **Run workflow**

## Устранение проблем

### Сайт не обновляется
- Проверьте, что workflow завершился успешно
- Убедитесь, что в настройках Pages выбран источник `GitHub Actions`
- Подождите несколько минут (GitHub Pages может обновляться с задержкой)

### 404 ошибка или изображения не отображаются
- ✅ Base path настраивается автоматически - проверьте логи workflow, что он определён правильно
- ✅ Все пути к изображениям (начинающиеся с `/`) автоматически адаптируются
- ✅ Убедитесь, что файлы изображений находятся в папке `public/`
- ✅ Проверьте, что workflow завершился успешно и base path установлен корректно

### Ошибки сборки
- Проверьте логи в разделе **Actions**
- Убедитесь, что все зависимости установлены корректно
- Проверьте, что TypeScript компилируется без ошибок
